### JWT 보안 설계 — "로그아웃 전까지 로그인 유지"

**토큰 구조**

| 토큰 | 유효기간 | 저장 위치 | 용도 |
|------|----------|-----------|------|
| Access Token | **1시간** | 메모리 (JS 변수) | API 인증 헤더 (Bearer) |
| Refresh Token | **30일** | httpOnly + Secure + SameSite=Strict Cookie | Access Token 재발급 |

**핵심 메커니즘: Silent Refresh (자동 갱신)**
1. Access Token 만료 → FE가 자동으로 `/api/auth/refresh` 호출
2. BE가 Refresh Token 검증 → 새 Access Token + **새 Refresh Token** 발급 (Rotation)
3. 사용자는 만료를 인지하지 못함 → **로그아웃 전까지 끊김 없는 세션**
4. 30일간 미접속 시에만 자연 만료 → 재로그인 필요

**보안 대책**

| 위협 | 대응 |
|------|------|
| **Refresh Token 탈취** | Refresh Token Rotation — 사용 시 즉시 폐기 후 새 토큰 발급. 이미 사용된 토큰 재사용 감지 시 해당 사용자 전체 세션 무효화 |
| **XSS** | Access Token은 메모리 전용 (localStorage 저장 금지). Refresh Token은 httpOnly Cookie로 JS 접근 차단 |
| **CSRF** | SameSite=Strict + CSRF Token 이중 방어 |
| **다중 기기** | Redis에 기기별 Refresh Token 관리. 기기 목록 조회 + 개별 로그아웃 가능 |
| **강제 로그아웃** | Redis에서 Refresh Token 삭제 → 즉시 무효화 |
| **비밀번호 변경** | 전체 기기 Refresh Token 일괄 폐기 |

**결론**: Access 1시간 + Refresh 30일 + Silent Refresh + Rotation 조합으로 보안과 UX를 모두 확보. 업계 표준(OAuth 2.0 BCP)을 준수하며, 30분/7일 방식보다 UX가 개선되고 Rotation으로 보안은 오히려 강화됨.

## 10. AI 일정 비서 (프리미엄 구독 전용)

> 타임트리에 없는 핵심 차별화 기능. 구독 사용자에게 Claude API를 활용한 AI 비서 제공.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| AI 일정 생성 | 자연어 입력 → 이벤트 자동 파싱 ("다음 주 수요일 3시에 치과 예약" → 이벤트 생성) | P1 |
| 일정 요약 | 오늘/이번 주/이번 달 일정을 자연어로 요약 브리핑 | P1 |
| 충돌 감지 & 제안 | 일정 겹침 감지 + 대안 시간 자동 제안 | P1 |
| 빈 시간 탐색 | "이번 주 멤버 전원이 비는 시간 찾아줘" → 공통 가능 시간 추천 | P2 |
| 일정 리마인드 | 맥락 기반 알림 ("내일 치과인데 보험증 챙기세요") | P2 |
| 반복 패턴 학습 | 사용자 일정 패턴 분석 → "매주 금요일 저녁 약속이 많네요" 인사이트 제공 | P2 |

### AI 기술 구현

| 항목 | 상세 |
|------|------|
| **AI 모델** | Claude API (Anthropic) |
| **호출 방식** | BE에서 서버사이드 호출 (API Key 노출 방지) |
| **요금 구조** | 구독자에게만 AI 기능 활성화. API 호출 비용은 서비스 운영비에 포함 |
| **Rate Limit** | 구독자당 일 50회 AI 요청 제한 (남용 방지) |
| **데이터 범위** | 해당 사용자의 캘린더 이벤트만 컨텍스트로 전달 (개인정보 최소화) |
| **응답 형식** | Structured Output → FE에서 이벤트 생성/수정 액션으로 변환 |

## Phase 1 (웹 MVP) 핵심 범위 요약

```
인증(이메일 + 카카오/구글/애플) → 캘린더 생성/관리 → 멤버 초대(카톡/이메일)
→ 이벤트 CRUD(반복/알림) → 월간 뷰 + 주간 시간대 뷰 → 이벤트 댓글
→ 실시간 동기화(WebSocket) → 변경 알림
→ [프리미엄] AI 일정 비서(자연어 생성/요약/충돌 감지)
```

**MVP 목표**: "함께 쓰는 캘린더"의 핵심 경험을 웹에서 완성

## 기술 스택

### FE (프론트엔드)

| 항목 | 기술 | 선정 이유 |
|------|------|-----------|
| **프레임워크** | Next.js 15 (App Router) | SSR/SSG 지원, Cloudflare Pages 배포 호환 |
| **언어** | TypeScript 5 | 타입 안전성, DX 향상 |
| **스타일** | CSS Modules | 전역 전략 준수, 컴포넌트 스코프 격리 |
| **상태 관리** | Zustand | 경량, 보일러플레이트 최소, 실시간 상태에 적합 |
| **HTTP 클라이언트** | Axios + React Query (TanStack Query) | 캐싱, 자동 재시도, Silent Refresh 연계 |
| **실시간 통신** | SockJS + STOMP.js | Spring WebSocket과 호환, 재연결 자동 처리 |
| **캘린더 UI** | 직접 구현 (커스텀) | 월간/주간 시간대 뷰를 브랜드 디자인에 맞게 자유 제어 |
